# Model Card (v1)

## Назначение

Ранжирование H3-зон Москвы для **исследовательской** приоритизации размещения банкоматов ВТБ на основе агрегированных транзакционных и POI-признаков.

## Версионирование

`model_version` в API — строка вида `YYYYMMDD-HHMMSS-<sha256-короткий>` после каждого `ingest/retrain`.

## Обучающие данные

- Источник: `dataset_final.csv` (агрегаты, без персональных идентификаторов клиентов).
- Разметка: **псевдо-лейблы** из правил сценариев (см. `METHODOLOGY.md`).

## Алгоритмы

- **Supervised**: `RandomForestClassifier` (часто) или fallback `RandomForestRegressor` на `heuristic_score`.
- **Unsupervised**: `KMeans`.

## Метрики мониторинга качества (offline)

- `spearman_ml_vs_heuristic`
- `kendall_ml_vs_heuristic` (если scipy доступен)
- `roc_auc_holdout_pseudo` по псевдо-лейблам
- `coverage_at_topk_pseudo_positive`
- `kendall_bootstrap_std`

## Смещения и риски

- Данные только по Москве → **не переносить** на другие города без переобучения.
- Агрегаты могут отражать бизнес-перекосы (MCC, сезонность) — отсутствуют в финальном CSV.
- Псевдо-разметка следует за эвристикой → модель не «открывает истину», а **сглажает/комбинирует** нелинейно те же сигналы + дополнительные зависимости.

## Этика / PII

Используются только агрегаты по зонам. Персональные данные клиентов **не хранятся** в сервисе v1.

## Рекомендации по применению

Использовать как **один из факторов** вместе с юнит-экономикой, юридическими ограничениями, полевым аудитом помещений.
