Анализ предоставленных данных

Факторы, влияющие на потенциал размещения банкомата в зоне

👤
Клиентские факторы
  • • unique_customers — число уникальных клиентов в зоне
  • • sum_per_customer — средние траты на клиента
  • • transactions_per_customer — частота транзакций
  • • atm_active_customers — клиенты, уже использующие банкоматы
  • • unique_mcc — разнообразие категорий трат (поведение)
🗺️
Географические факторы
  • • metro_count — станции метро рядом (трафик)
  • • mall_count — ТЦ / торговые точки
  • • university_count — университеты (аудитория)
  • • H3-индекс (res.9) — гексагональная ячейка ~0.1 км²
  • • unique_time_buckets — активность в разное время суток
🏦
Инфраструктурные факторы
  • • vtb_atm_count — уже размещённые банкоматы ВТБ
  • • vtb_office_count — отделения ВТБ в зоне
  • • competitor_atm_count — банкоматы конкурентов (Сбер, Альфа)
  • • total_sum — суммарный объём транзакций
  • • total_count — общее число операций
📊
Поведенческие факторы
  • • avg_transaction — средний чек (тип аудитории)
  • • avg_std — волатильность трат (нерегулярность)
  • • mcc_code — категория точки (рестораны, АЗС, …)
  • • datetime_id — временные бакеты (0=ночь, 3=вечер)
  • • atm_activity — целевая метка (есть ли банкомат)

Логика оценки и ранжирования зон

5-шаговая методология от данных до рекомендации

1
Агрегация данных по H3-зонам
Группировка 4,15 млн транзакций по h3_index: unique_customers, total_sum, total_count, avg_transaction, unique_mcc, unique_time_buckets. Результат — 8 154 строки-признаки.
2
Подключение геоинфраструктурных слоёв
Join с внешними данными: банкоматы ВТБ (N=n), отделения ВТБ, банкоматы конкурентов (Сбер, Альфа-Банк), станции метро, ТЦ, университеты → счётчики на H3-ячейку.
3
Нормализация признаков (MinMaxScaler) → Взвешенный скоринг
Каждый признак нормируется в [0,1]. Итоговый ATM Score = 0.30·norm(unique_customers) + 0.25·norm(total_sum) + 0.20·norm(total_count) + 0.10·norm(unique_mcc) + 0.08·norm(metro_count+mall_count) − 0.07·norm(vtb_atm_count).
4
Фильтрация кандидатов и ML-валидация
Оставляем зоны: atm_activity=0, ATM Score > 75-й перцентиль, vtb_atm_count=0. Логистическая регрессия / Gradient Boosting на обучающих зонах (atm_activity=1) для вероятности P(банкомат нужен).
5
Итоговое ранжирование и шорт-лист
Сортировка по ATM Score (убывание). Топ-10 зон → шорт-лист. Дополнительно: исключение зон с >2 банкоматами конкурентов в непосредственной близости (борьба за долю) или включение (если высокий спрос).

💰
Ценность для банка
  • ✅ Рост комиссионного дохода с новых точек
  • ✅ Снижение стоимости неверно установленных банкоматов
  • ✅ Увеличение лояльности клиентов (доступность сервиса)
  • ✅ Перехват аудитории у конкурентов в высокопотенциальных зонах
  • ✅ Сокращение цикла принятия решения: с недель до часов
🚀
Идеи усовершенствования
  • 🔄 Оптимизация: выявление банкоматов ВТБ с падающей активностью для релокации
  • 🏗️ Прогноз спроса: интеграция данных о строительстве ЖК, ТЦ, БЦ (РГИС, Яндекс.Карты API)
  • 📡 Real-time мониторинг: динамическое обновление скорингов при изменении трафика
  • 🤖 ML 2.0: LGBM/XGBoost с временными признаками и авторанжировкой весов
⚠️
Ограничения и риски
  • ⚡ Данные — срез во времени (datetime_id 0-3), сезонность не учтена
  • ⚡ Отсутствие данных об аренде/собственности помещений
  • ⚡ Корреляция: клиенты ВТБ ≠ весь пешеходный трафик зоны
